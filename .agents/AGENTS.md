# Reglas del Proyecto (sd_supplychain)

## Estándares de Commits
Formato obligatorio para todos los commits en este proyecto:
`Tipo (v.XXX) : Descripción`

Reglas:
1. El `Tipo` debe estar en español (Ej: Mejora, Fix, Feature, Seguridad).
2. La versión `(v.XXX)` debe ir entre paréntesis después del tipo.
3. Todo el texto (descripciones, variables, commits) debe estar en español.

Ejemplo válido: `Mejora (v.1.01) : Añadir versión junto al logo principal en la barra lateral`
